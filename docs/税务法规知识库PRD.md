## 1. 产品概述

### 1.1 产品背景

随着国家税法体系日益复杂，企业财务人员、税务师、审计人员等需要处理大量法规文档。传统关键词搜索无法满足以下需求：

- **语义理解**：无法理解"小规模纳税人增值税优惠政策"与"增值税小规模纳税人减免规定"的关联
- **关系推理**：无法展示"企业所得税率25%"与"高新技术企业15%优惠税率"的关系
- **实体定位**：无法快速定位某项优惠政策适用于哪些纳税人类型

本产品旨在构建一个基于 LightRAG 的税务法规知识库，结合大语言模型能力，实现智能问答和知识图谱管理。

### 1.2 产品目标

| 目标 | 描述 | 成功指标 |
|------|------|---------|
| 智能问答 | 基于法规知识的自然语言问答 | 问答准确率 > 85% |
| 自定义实体 | 支持用户自定义税务实体类型 | 用户可配置 Schema |
| 图谱管理 | 可视化知识图谱，支持增删改 | 操作响应 < 2s |
| 文档导入 | 支持 PDF/Word/Excel 格式 | 支持主流格式 |

### 1.3 目标用户

| 用户角色 | 使用场景 | 主要功能 |
|---------|---------|---------|
| 税务专员 | 查询政策、准备税务申报 | 智能问答、法规检索 |
| 税务主管 | 审核税务方案、培训下属 | 知识图谱查看、批量导入 |
| 系统管理员 | 系统配置、用户管理 | 实体 Schema 管理、权限管理 |

### 2.2 核心功能模块

#### 2.2.1 文档管理模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 文件上传 | 支持 PDF、Word、Excel 格式，限制单文件 50MB | P0 |
| 文档解析 | 提取文本内容，支持表格结构识别 | P0 |

#### 2.2.2 实体 Schema 管理模块

**功能描述**：用户可自定义税务领域的实体类型和关系类型，导入文档时按自定义 Schema 进行抽取。

| 功能 | 描述 | 优先级 |
|------|------|--------|
| Schema 模板 | 提供预设的税务领域 Schema 模板 | P0 |
| Schema 查看 | 查看当前激活的实体类型和关系类型 | P0 |
| Schema 新增 | 新增自定义实体类型（名称、描述、属性） | P0 |
| Schema 编辑 | 修改现有实体类型定义 | P0 |
| Schema 删除 | 删除自定义实体类型（需确认无关联数据） | P1 |
| 关系类型管理 | 新增/编辑/删除关系类型定义 | P0 |

**实体 Schema 定义结构**：

```json
{

    },
    {
      "name": "税目",

    {
      "name": "税率",
      "description": "税额计算比例或金额",
```

**问答能力示例**：

| 用户问题 | 系统回答 |
|---------|---------|
| "小规模纳税人增值税税率是多少？" | 2026年小规模纳税人增值税税率为1%（优惠期间），适用季度销售额30万元以下的小规模纳税人。依据：《财政部 税务总局公告2023年第1号》 |

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 |
|------|------|
| 文档导入 | 单个 PDF（10MB）解析 < 30s |
| 问答响应 | 首 token 延迟 < 3s（LLM 推理） |
| 图谱加载 | 500 节点以内 < 2s |
| 并发用户 | 支持 50 用户同时在线 |

### 3.2 安全需求

- 用户密码加密存储（bcrypt）
- 敏感操作需登录验证
- 管理员操作需二次确认
- 私有化部署，数据不出网

### 3.3 部署需求

- **部署方式**：私有化部署
- **部署环境**：Linux 服务器（推荐 CentOS 7+ / Ubuntu 20.04+）
- **硬件要求**：
  - CPU: 8 核+
  - 内存: 32GB+
  - 硬盘: 500GB+
  - GPU: NVIDIA 16GB+（如使用本地 LLM）
